Hydrologic Micro Services, HMS, is a watershed modeling system comprised of data provisioning and modeling components and workflows to facilitate environmental modeling. The components are available as web services, desktop libraries, and through a web-based graphical user interface. The target users for HMS are hydrological and water quality modelers and analysts. HMS users can use the system components and workflows directly or build their own workflows by linking the components.

HMS functionality will be limited when using the Internet Explorer web browser. We recommend using the latest version of Google Chrome, Microsoft Edge, or Mozilla Firefox.

Structure of HMS web app

Components

An HMS component is a module that encapsulates a set of related functions and data representing a physical process in hydrology or fate and transport of a contaminant in water. HMS components are presented as web services, software libraries, and web forms.

Work Flows

An HMS workflow is an orchestrated pattern of executing components to simulate movement and storage of water and the consequential effects on fate and transport of contaminants in water.

  • Precipitation Data Extraction
  • Precipitation Comparison
